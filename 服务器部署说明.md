# Harvest IT 服务器部署说明

## 在 VirtualBox 虚拟机中部署服务器

### 步骤1：设置虚拟机

1. 创建新虚拟机
   - 名称：HarvestIT-Server
   - 类型：Linux
   - 版本：Ubuntu (64-bit)
   - 内存：至少 1GB（建议 2GB）
   - 硬盘：20GB（动态分配）

2. **重要：网络设置**
   - 网络适配器1：**桥接网卡（Bridged Adapter）**
   - 这样虚拟机可以获得局域网 IP，其他电脑可以访问

3. 安装 Ubuntu Server 22.04 LTS

### 步骤2：在虚拟机中安装 Node.js

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node --version
npm --version

# 安装 PM2（进程管理器，让服务器后台运行）
sudo npm install -g pm2
```

### 步骤3：上传服务器文件

将以下文件上传到虚拟机：
- `server.js`
- `package.json`

可以使用以下方法：
1. 使用共享文件夹
2. 使用 scp 命令
3. 直接在虚拟机中创建文件

### 步骤4：安装依赖并启动

```bash
# 进入项目目录
cd ~/harvestit-server  # 或你的项目目录

# 安装依赖
npm install

# 测试运行
node server.js

# 如果看到 "🚀 Harvest IT Server running..." 说明成功
# 按 Ctrl+C 停止

# 使用 PM2 后台运行（推荐）
pm2 start server.js --name harvestit-server
pm2 save
pm2 startup  # 设置开机自启，按提示执行命令
```

### 步骤5：配置防火墙

```bash
# Ubuntu 使用 ufw
sudo ufw allow 3000/tcp
sudo ufw enable
```

### 步骤6：获取服务器 IP 地址

```bash
# 查看 IP 地址
ip addr show
# 或者
hostname -I
```

假设服务器 IP 是 `192.168.1.100`，那么 API 地址就是 `http://192.168.1.100:3000`

### 步骤7：测试服务器

在你的电脑浏览器中访问：
```
http://192.168.1.100:3000/api/health
```

应该看到：
```json
{"status":"ok","message":"Harvest IT Server is running"}
```

## 学生电脑设置

1. 确保学生电脑和服务器在同一局域网
2. 打开游戏页面
3. 首次打开会提示输入服务器地址
4. 输入：`http://192.168.1.100:3000`（改成你的服务器 IP）
5. 或者点击"设置服务器"按钮手动设置

## 常用命令

```bash
# 查看服务器状态
pm2 status

# 查看日志
pm2 logs harvestit-server

# 重启服务器
pm2 restart harvestit-server

# 停止服务器
pm2 stop harvestit-server

# 查看数据库（可选）
sqlite3 harvestit.db
SELECT * FROM records ORDER BY score DESC LIMIT 10;
.quit
```

## 故障排查

1. **无法连接服务器**
   - 检查服务器是否运行：`pm2 status`
   - 检查防火墙：`sudo ufw status`
   - 检查 IP 地址：`hostname -I`
   - 检查网络：确保在同一局域网

2. **端口被占用**
   - 修改 `server.js` 中的 `PORT` 变量
   - 或使用环境变量：`PORT=3001 node server.js`

3. **数据库错误**
   - 检查文件权限
   - 删除 `harvestit.db` 重新创建

## 注意事项

- 确保所有电脑在同一局域网
- 服务器 IP 可能会变化，如果变化了需要重新设置
- 建议在虚拟机中固定 IP 地址（可选）
- 数据库文件 `harvestit.db` 会保存在服务器目录中
